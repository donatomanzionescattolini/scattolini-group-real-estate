# Mobile App — PWA + iOS/Android (Capacitor)

The Scattolini Group site ships in three forms from a **single React codebase**:

1. **Web** — the normal site (Vercel).
2. **Installable PWA** — visitors can "Add to Home Screen" and run it offline-capable, full-screen.
3. **Native apps** — real iOS and Android apps (via [Capacitor](https://capacitorjs.com)) submittable to the **App Store** and **Google Play**.

There is no separate mobile codebase — the native apps load the same built web app (`dist/`) inside a native shell.

## 1. PWA (works today, no stores required)

Configured with [`vite-plugin-pwa`](https://vite-pwa-org.netlify.app/) in [`vite.config.ts`](vite.config.ts).

- Generates a service worker (`dist/sw.js`) + web manifest (`dist/manifest.webmanifest`) on every `npm run build`.
- Precaches the app shell; runtime-caches S3 images and Google Fonts.
- Icons live in `public/` (`pwa-192x192.png`, `pwa-512x512.png`, `pwa-maskable-512x512.png`, `apple-touch-icon.png`).

**It deploys automatically with the normal Vercel build** — nothing extra to do. After deploy, visiting the site on a phone offers "Add to Home Screen" (Android/Chrome shows an install prompt; iOS Safari → Share → Add to Home Screen).

> iOS does **not** allow PWAs in the App Store — for an App Store listing you need the native build below.

## 2. Native apps (Capacitor)

### Prerequisites

| Platform | Needs |
|---|---|
| **Android** | [Android Studio](https://developer.android.com/studio) + JDK 17. (None of these are required just to develop the web app.) |
| **iOS** | **macOS** + [Xcode](https://developer.apple.com/xcode/) + CocoaPods (`sudo gem install cocoapods`). iOS **cannot** be built on Windows/Linux. |

The `android/` and `ios/` folders are committed Capacitor projects. Build artifacts (`build/`, `Pods/`, the copied `public/` web assets) are git-ignored and regenerated by `cap sync`.

### Day-to-day workflow

Any time the web app changes, rebuild and copy it into the native shells:

```bash
npm run build && npx cap sync
```

Then open the platform IDE:

```bash
npm run mobile:android   # build + sync + open Android Studio
npm run mobile:ios       # build + sync + open Xcode   (macOS only)
```

From there, press **Run** to launch on a simulator/emulator or a connected device.

### App icons & splash screens

Source art is generated from the company logo:

```bash
npm run icons                 # regenerates assets/ + public/ icons via sharp
npx capacitor-assets generate --iconBackgroundColor "#0E2D2F" --splashBackgroundColor "#0E2D2F"
```

- `npm run icons` builds the PWA icons (`public/`) and the Capacitor source images (`assets/icon.png`, `assets/splash.png`, `assets/splash-dark.png`).
- `capacitor-assets generate` fans those into every Android mipmap and iOS asset-catalog size.

## 3. Publishing

> These steps require **your** developer accounts and signing credentials — they can't be automated from here. Each store also runs a review (typically 1–3 days).

### Google Play (Android)

1. Create a **[Google Play Console](https://play.google.com/console)** account ($25 one-time).
2. In Android Studio: **Build → Generate Signed Bundle / APK → Android App Bundle (.aab)**. Create an upload keystore the first time and **back it up** — losing it means you can't update the app.
3. In Play Console: create the app, fill the store listing (title, description, screenshots, feature graphic, privacy policy URL), set content rating & data-safety form.
4. Upload the `.aab` to the **Production** (or Internal testing) track and roll out.

### App Store (iOS) — requires a Mac

1. Enroll in the **[Apple Developer Program](https://developer.apple.com/programs/)** ($99/year).
2. In Xcode: set the **Signing Team** under *Signing & Capabilities*, bump the version/build, then **Product → Archive**.
3. In the Organizer, **Distribute App → App Store Connect**.
4. In **[App Store Connect](https://appstoreconnect.apple.com)**: create the app record, fill the listing (description, keywords, screenshots for each device size, privacy details), attach the build, and submit for review.

### Bumping the version (do this for every store release)

- **Android:** `android/app/build.gradle` → `versionCode` (integer, must increase) and `versionName` (e.g. `1.0.1`).
- **iOS:** in Xcode target → **General** → *Version* and *Build*.

## Important: contact forms

All in-app forms post to **Web3Forms** and need `VITE_WEB3FORMS_KEY` set at **build time** (see [`.env.example`](.env.example)). Because the native apps bundle whatever `dist/` was built with, make sure the key is present in your local `.env` (or CI env) **before** `npm run build && npx cap sync`, otherwise the bundled app will ship without working forms.
